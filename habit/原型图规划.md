# 习惯App原型图规划

## Why

根据P0 PRD定义的关键页面和功能需求，规划完整的原型图数量、结构内容和交互设计。

## What Changes

- 明确原型图总数量和命名规范
- 定义每张原型图的内容结构
- 规划页面间导航关系
- 确定交互模式和状态覆盖

## Impact

- Affected specs: P0 PRD全文
- Affected code: 原型设计文件

***

## 原型图清单

### 导航结构

采用**底部Tab导航 + 顶部标题栏**的经典移动端架构：

```
┌─────────────────────────┐
│      顶部标题栏          │  ← 页面标题 + 可选操作按钮
├─────────────────────────┤
│                         │
│       主内容区           │  ← 页面核心内容
│                         │
├─────────────────────────┤
│ 今日 │ 标签 │ 统计 │ 激励 │ 设置 │  ← 底部Tab导航（5个）
└─────────────────────────┘
```

### 排序操控按钮

今日页面和标签页面顶部提供排序操控入口：

```
┌─────────────────────────────────┐
│ ← 今日清单        🔍  ⋮  📅   │  ← ⋮ 三竖点按钮
├─────────────────────────────────┤
```

点击 **⋮** 三竖点按钮，展开排序操作面板：

```
┌─────────────────────────────────┐
│ 排序方式                    ✕   │
├─────────────────────────────────┤
│                                 │
│  ○ 自定义（手动拖拽）          │  ← 默认选中
│  ○ 名称（A-Z）                 │
│  ○ 创建时间                    │
│  ○ 完成进度                    │
│                                 │
│  点住右侧 ☰ 可拖拽排序          │  ← 引导提示
│                                 │
└─────────────────────────────────┘
```

**交互说明**：

- 点击 **⋮** 按钮 → 展开排序面板
- 选择排序规则 → 应用到当前列表
- 选择"自定义" → 进入拖拽模式，显示 ☰ 拖拽手柄
- 拖拽模式：长按 ☰ → 拖动调整顺序 → 松开放置

```
│  ┌────────────┐                 │

│  └────────────┘                 │

│  名称 [____________________]    │
│                                 │
│  颜色 [●●●●●●●●]              │  ← 颜色选择器
│                                 │
│  标签 [多选: 健康/运动/学习...] │
│                                 │
│  频率 [每日 ▼]                 │
│                                 │
│  每日目标 [1] 次               │
│                                 │

│                                 │
│  备注 [____________________]   │
│                                 │
└─────────────────────────────────┘
```

**字段说明**：

| 字段   | 类型   | 必填 | 说明           |
| ---- | ---- | -- | ------------ |
| 图标   | 选择器  | 是  | 从预置图标库选择     |
| 名称   | 输入框  | 是  | 最长20字符       |
| 颜色   | 选择器  | 是  | 8种预设低饱和色     |
| 标签   | 多选   | 否  | 预置标签库        |
| 频率   | 下拉   | 是  | 每日/每周/每月/自定义 |
| 每日目标 | 数字   | 是  | 1-99次        |
| 开始日期 | 日期选择 | 是  | 默认今天         |
| 备注   | 文本域  | 否  | 最长100字符      |

**交互**：

- 点击图标 → 展开图标选择网格
- 点击颜色 → 选中并高亮
- 点击标签 → 多选切换
- 点击频率 → 展开选项菜单
- 点击保存 → 验证并保存，返回今日清单

## 关键设计决策

1. **底部Tab固定导航**：5个核心入口（今日/标签/统计/激励/设置），符合移动端习惯
2. **今日清单为主入口**：用户打开App直接看到今日任务
3. **卡片式列表**：信息层级清晰，触控区域足够大(48px)
4. **⋮ 排序操控按钮**：今日和标签页顶部提供三竖点按钮，点击展开排序面板
5. **☰ 拖拽手柄**：选择"自定义"排序时，习惯卡片左侧显示拖拽手柄
6. **空状态引导**：每个空状态都提供明确的创建入口
